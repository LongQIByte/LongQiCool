# 强化学习基础

强化学习（Reinforcement Learning, RL）是机器学习的一个分支，通过智能体（Agent）与环境（Environment）的交互来学习最优策略。智能体通过试错的方式，在获得奖励或惩罚的反馈中不断优化行为。

## 基本概念

### 马尔可夫决策过程（MDP）

强化学习问题通常建模为马尔可夫决策过程（Markov Decision Process, MDP），包含以下要素：

#### 状态（State）

状态是环境在某个时刻的完整描述，包含了智能体做出决策所需的所有信息。

**例子**：

- **围棋游戏**：状态可以是棋盘上所有棋子的位置（19×19的矩阵，每个位置表示黑子、白子或空）
- **机器人控制**：状态可能包括机器人的关节角度、速度、位置坐标等（如 [θ₁, θ₂, ..., θₙ, v₁, v₂, ..., vₙ, x, y, z]）
- **自动驾驶**：状态包括车辆位置、速度、周围车辆位置、交通灯状态、道路信息等

状态可以是**离散的**（如棋盘位置）或**连续的**（如机器人关节角度），也可以是**部分可观测的**（智能体只能看到环境的一部分信息）。

#### 动作（Action）

动作是智能体在某个状态下可以执行的操作，是智能体影响环境的方式。

**例子**：

- **围棋**：动作是在棋盘上落子的位置（19×19=361个可能的动作，或加上"pass"共362个）
- **机器人控制**：动作是发送给各个关节的力矩或目标角度（连续动作空间，如 [τ₁, τ₂, ..., τₙ]）
- **自动驾驶**：动作可能是转向角度和加速度（连续动作，如 [steering_angle, acceleration]）
- **推荐系统**：动作是向用户推荐哪个商品或内容（离散动作空间）

动作空间可以是**离散的**（有限个选择）或**连续的**（在某个范围内任意值）。

#### 奖励（Reward）

奖励是执行动作后环境给出的即时反馈信号，告诉智能体这个动作的好坏。

**例子**：

**围棋**：奖励设计非常稀疏，只有在游戏结束时才给出奖励。赢得比赛获得 +1，输掉比赛获得 -1，中间的所有步骤都是 0。这种稀疏奖励设计使得智能体需要学习长期策略，不能只看眼前利益。

**机器人行走**：每向前移动一步获得 +1，摔倒获得 -100，保持平衡获得 +0.1。这种奖励设计鼓励机器人既要前进（完成任务），又要保持稳定（避免失败）。

**自动驾驶**：安全到达目的地获得 +1000，发生碰撞获得 -1000，违反交通规则获得 -100，平稳驾驶获得 +0.1。这种设计将安全性放在首位（碰撞惩罚很大），同时鼓励遵守规则和平稳驾驶。

奖励设计是强化学习的关键，需要平衡**即时奖励**（鼓励短期行为）和**长期回报**（最终目标）。智能体的目标是最大化**累积奖励**（Return），即未来所有奖励的加权和。

#### 策略函数（Policy）

策略是从状态到动作的映射函数，决定智能体在某个状态下应该选择什么动作。

**例子**：

- **确定性策略**：给定状态，总是选择同一个动作
  - 如：在围棋的某个局面下，总是选择胜率最高的落子位置
  - 数学表示：a = π(s)，直接输出动作
- **随机性策略**：给定状态，输出动作的概率分布
  - 如：在某个游戏状态下，80%概率选择动作A，20%概率选择动作B（用于探索）
  - 数学表示：π(a|s) = P(A=a|S=s)，输出动作概率

**实际例子**：

- **训练初期**：策略可能是随机选择动作（探索）
- **训练后期**：策略学会了在危险状态选择安全动作，在有利状态选择进攻动作
- **AlphaGo**：策略网络学习在每种棋盘局面下，哪些位置更可能获胜

策略可以是**表格形式**（状态-动作查找表）或**函数形式**（神经网络参数化），后者可以处理高维状态空间。

#### 价值函数（Value Function）

价值函数评估状态或状态-动作对的好坏，表示从当前状态开始，遵循某个策略能获得的**期望累积奖励**，注意这里的状态就是上面提到的state。

**例子**：

1. **状态价值函数 V(s)**：评估状态的好坏
   - **围棋**：V(s) 表示从当前棋盘局面开始，使用当前策略能获胜的概率（接近1表示优势，接近0表示劣势）
   - **游戏**：V(s) 表示从当前游戏画面开始，能获得多少总分
   - **机器人**：V(s) 表示从当前位置和姿态开始，能完成任务的期望奖励

2. **动作价值函数 Q(s,a)**：评估在状态s下执行动作a的好坏
   - **Atari 游戏**：Q(s, "开火") 表示在当前画面下开火，能获得多少期望分数
   - **机器人控制**：Q(s, [τ₁, τ₂]) 表示在当前状态下施加这些力矩，能获得多少期望奖励

**直观理解**：

- 如果 Q(s, a₁) = 100，Q(s, a₂) = 50，说明在状态s下，动作a₁比a₂更好
- 价值函数帮助智能体做决策：选择价值最高的动作
- 价值函数考虑了**长期回报**，而不仅仅是即时奖励

**实际应用**：

- **训练过程**：价值函数不断更新，越来越准确地预测未来回报
- **决策过程**：使用 Q(s,a) 选择最优动作，a\* = argmax_a Q(s,a)
- **策略评估**：通过价值函数判断策略是否在改进

#### 如何学习价值函数：TD（时序差分）学习

价值函数不是预先知道的，需要通过与环境交互来学习。**TD（Temporal Difference，时序差分）学习**是学习价值函数的核心方法。

**核心思想**：利用"当前估计"和"下一步估计"之间的差异来更新价值函数，而不需要等到整个回合结束。

**TD 误差（TD Error）**：

$$
\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)
$$

其中：

- $r_t$：即时奖励
- $\gamma$：折扣因子（0-1之间，表示未来奖励的重要性）
- $V(s_{t+1})$：下一状态的价值（当前估计）
- $V(s_t)$：当前状态的价值（当前估计）

**更新规则**：

$$
V(s_t) \leftarrow V(s_t) + \alpha \delta_t
$$

其中 $\alpha$ 是学习率。

**为什么叫"时序差分"**：

- **时序**：利用时间序列上的相邻状态（$s_t$ 和 $s_{t+1}$）
- **差分**：使用两个时刻的估计值之间的差异来更新

**TD 学习的优势**：

1. **在线学习**：不需要等到回合结束，每一步都可以更新
2. **样本效率高**：每一步的经验都立即用于学习
3. **自举（Bootstrap）**：使用自己的估计来更新自己（$V(s_{t+1})$ 也是估计值）

**例子**：

- 在机器人抓取任务中，如果 $V(s_t) = 85.3$（当前估计），执行动作后获得奖励 $r_t = -0.1$，到达新状态 $V(s_{t+1}) = 85.5$（估计值）
- TD 误差：$\delta_t = -0.1 + 0.9 \times 85.5 - 85.3 = -0.95$（假设 $\gamma = 0.9$）
- 更新：$V(s_t) \leftarrow 85.3 + 0.1 \times (-0.95) = 85.205$（假设 $\alpha = 0.1$）
- 价值函数逐渐变得更准确

**TD vs 其他方法**：

- **蒙特卡洛方法**：需要等到回合结束，使用实际累积奖励 $G_t$ 来更新
- **TD 方法**：每一步都可以更新，使用 $r_t + \gamma V(s_{t+1})$ 作为目标（更快，但可能有偏差）
- **TD 是强化学习的核心**：大多数现代算法（DQN、DDPG、PPO 等）都使用 TD 学习来更新价值函数

#### 价值函数与策略的区别

**核心区别**：

1. **价值函数是"评估器"，策略是"执行器"**
   - 价值函数 Q(s,a) 告诉你"在状态s下，动作a有多好"，但它本身不直接执行动作
   - 策略 π(a|s) 直接告诉你"在状态s下，应该选择什么动作"
   - 类比：价值函数像"评分系统"（给每个选项打分），策略像"决策系统"（直接做出选择）

2. **离散动作空间 vs 连续动作空间**
   - **离散动作空间**（如围棋、Atari游戏）：
     - 理论上，如果 Q(s,a) 对所有动作都准确，确实可以通过 argmax_a Q(s,a) 直接选择最优动作
     - 但策略仍然重要：需要探索（ε-greedy）、处理不确定性、提供随机性
     - 例如：DQN 使用 Q 网络选择动作，但仍需要 ε-greedy 策略进行探索
   - **连续动作空间**（如机器人控制、自动驾驶）：
     - **这是关键**：动作空间是无限的（如转向角度可以是 0° 到 360° 之间的任意值）
     - 无法枚举所有可能的动作来计算 argmax，必须通过策略网络直接输出动作值
     - 例如：DDPG 的 Actor 网络直接输出连续动作 [τ₁, τ₂, ...]，而不是通过 Q 函数搜索

3. **探索与利用的平衡**
   - 价值函数通常用于"利用"（选择已知最好的动作）
   - 策略可以设计为"探索"（尝试新动作，发现更好的策略）
   - 例如：策略可以是随机性的 π(a|s)，在训练初期探索更多，后期逐渐确定化

4. **实际应用中的结合**：根据主导方式的不同，强化学习算法可以分为三类：
   - **Value-based 方法**（如 DQN）：**价值函数主导**
     - 主要学习价值函数 $Q(s,a)$，策略通过 $\pi(s) = \arg\max_a Q(s,a)$ 隐式定义
     - **策略更新方式**：通过更新价值函数来间接更新策略。价值函数更准确后，argmax 选择的动作自然就更好
     - 优点：样本效率高，训练稳定
     - 缺点：只适用于离散动作空间，无法处理连续动作
     - 例子：DQN 学习 Q 网络，策略就是"选择 Q 值最大的动作"
   - **Policy-based 方法**（如 REINFORCE）：**策略函数主导**
     - 直接学习策略函数 $\pi_\theta(a|s)$，可能不使用价值函数
     - **策略更新方式**：使用策略梯度直接更新策略参数。通过实际获得的累积奖励（或优势估计）来调整策略，增加好轨迹的概率，减少坏轨迹的概率
     - 优点：可以处理连续动作空间，策略表达灵活
     - 缺点：样本效率低，方差大，训练不稳定
     - 例子：REINFORCE 直接根据累积奖励更新策略，不使用价值函数
   - **Actor-Critic 方法**（如 DDPG、PPO）：**两者结合**
     - 同时学习策略网络（Actor）和价值网络（Critic）
     - **策略更新方式**：价值函数（Critic）评估动作好坏，指导策略（Actor）更新。策略网络负责执行，价值网络负责评估和提供更新信号
     - 优点：结合两者优势，样本效率高，训练稳定，可处理连续动作
     - 缺点：需要同时训练两个网络，实现复杂
     - 例子：PPO 使用价值函数计算优势，然后用优势函数更新策略；DDPG 使用 Q 函数指导策略梯度更新

**总结**：价值函数告诉你"什么好"，策略告诉你"怎么做"。在连续动作空间中，策略是必需的；在离散动作空间中，策略提供了探索和随机性。实际应用中，两者往往协同工作，策略网络负责执行，价值网络负责评估和优化。

#### 综合例子：机器人抓取任务

让我们用一个完整的例子来理解 MDP 的所有要素如何协同工作。假设我们训练一个机械臂抓取桌面上的杯子。

**场景描述**：机械臂需要从初始位置移动到桌面，抓取杯子，然后放到指定位置。

**一个完整的决策过程**：

1. **状态（State）**：在时刻 $t$，状态 $s_t$ 包含：
   - 机械臂的关节角度：$[\theta_1=30°, \theta_2=45°, \theta_3=60°, \theta_4=90°]$
   - 末端执行器的位置：$[x=0.3m, y=0.2m, z=0.1m]$
   - 杯子的位置：$[x_{cup}=0.5m, y_{cup}=0.3m, z_{cup}=0.05m]$
   - 是否已抓取：`grasped=False`
   - 目标位置：$[x_{target}=0.7m, y_{target}=0.4m, z_{target}=0.1m]$

   完整状态向量：$s_t = [30°, 45°, 60°, 90°, 0.3, 0.2, 0.1, 0.5, 0.3, 0.05, False, 0.7, 0.4, 0.1]$

2. **策略（Policy）**：策略网络 $\pi(a|s_t)$ 接收状态 $s_t$，输出动作的概率分布或直接输出动作值。
   - 输入：状态 $s_t$（14维向量）
   - 输出：动作 $a_t = [\Delta\theta_1, \Delta\theta_2, \Delta\theta_3, \Delta\theta_4, grip] = [2°, -1°, 3°, -2°, 0.1]$
     - 前4个值：各关节的角度变化量（连续值）
     - 最后一个值：抓取力（0-1之间，0.1表示轻微抓取）

3. **动作（Action）**：执行策略输出的动作 $a_t$。
   - 机械臂各关节按照 $[2°, -1°, 3°, -2°]$ 调整角度
   - 末端执行器施加 0.1 的抓取力
   - 环境根据物理引擎计算新的状态

4. **奖励（Reward）**：环境给出即时奖励 $r_t$。
   - 如果成功抓取杯子：$+100$
   - 如果杯子掉落：$-50$
   - 如果到达目标位置并放下杯子：$+200$
   - 每移动一步：$-0.1$（鼓励快速完成任务）
   - 如果碰撞障碍物：$-20$

   假设这一步只是移动，没有抓取，奖励 $r_t = -0.1$

5. **状态转移**：环境更新到新状态 $s_{t+1}$。
   - 新的关节角度：$[32°, 44°, 63°, 88°]$（根据动作更新）
   - 新的末端位置：$[0.32m, 0.21m, 0.12m]$（更接近杯子了）
   - 其他信息更新...

6. **价值函数（Value Function）**：评估当前状态和动作的好坏。
   - **状态价值 $V(s_t)$**：从当前状态 $s_t$ 开始，遵循当前策略 $\pi$，预期能获得的累积奖励
     - 假设 $V(s_t) = 85.3$，表示评估当前这个状态的价值为 85.3，即从这个状态开始，遵循当前策略预期能获得 85.3 的累积奖励
     - 类似于围棋中评估当前棋盘局面的价值（胜率）
   - **动作价值 $Q(s_t, a_t)$**：在状态 $s_t$ 下执行动作 $a_t$，然后遵循当前策略，预期能获得的累积奖励
     - 假设 $Q(s_t, a_t) = 85.5$，表示评估这个动作的价值为 85.5
     - 如果 $Q(s_t, a_t) > V(s_t)$，说明执行这个动作 $a_t$ 比遵循策略在当前状态下的预期表现更好，这个动作是有价值的、更好的选择
     - 如果 $Q(s_t, a_t) < V(s_t)$，说明这个动作不如策略的预期表现，应该避免

7. **策略更新**：根据不同的方法，策略更新的方式不同。

   **方式一：价值函数驱动（Value-based，如 DQN）**
   - 不直接更新策略，而是更新价值函数 $Q(s,a)$
   - 策略通过 $\pi(s) = \arg\max_a Q(s,a)$ 隐式定义
   - 更新过程（使用 TD 学习）：$Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha[r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t)]$
     - 其中 $r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t)$ 就是 **TD 误差**
     - 使用即时奖励 $r_t$ 和下一步的估计值 $\max_{a'} Q(s_{t+1}, a')$ 来更新当前估计
   - 价值函数更准确后，argmax 选择的动作自然就更好，策略间接得到改进
   - 在这个例子中：如果 $Q(s_t, a_1) = 85.5$，$Q(s_t, a_2) = 80.0$，策略会自动选择 $a_1$

   **方式二：策略函数驱动（Policy-based，如 REINFORCE）**
   - 直接更新策略参数 $\theta$，使用策略梯度
   - 更新过程：$\theta \leftarrow \theta + \alpha \nabla_\theta \log \pi_\theta(a_t|s_t) \cdot G_t$
   - 其中 $G_t$ 是实际获得的累积奖励（或优势估计）
   - 如果 $G_t$ 很大（好结果），增加选择 $a_t$ 的概率；如果 $G_t$ 很小（坏结果），减少概率
   - 在这个例子中：如果执行 $a_t$ 后获得高奖励，直接增加 $\pi_\theta(a_t|s_t)$ 的概率

   **方式三：Actor-Critic（价值函数指导策略更新，如 PPO、DDPG）**
   - Critic（价值网络）评估：$Q(s_t, a_t) = 85.5$，$V(s_t) = 85.3$
   - 优势函数：$A(s_t, a_t) = Q(s_t, a_t) - V(s_t) = 0.2$（正优势，说明这个动作比策略预期更好，是有价值的）
   - Actor（策略网络）更新：使用优势函数作为更新信号
     - 如果优势为正：$\theta \leftarrow \theta + \alpha \nabla_\theta \log \pi_\theta(a_t|s_t) \cdot A(s_t, a_t)$，增加选择类似动作的概率
     - 如果优势为负：减少概率
   - 价值函数提供更准确的更新信号（相比直接使用累积奖励），训练更稳定

**完整的学习循环**：

```python
初始化：策略网络 π_θ（随机参数），价值网络 V_φ（随机参数）

For episode = 1 to N:
    初始化状态 s_0（机械臂在初始位置）
    For t = 0 to T:
        1. 策略选择动作：a_t = π_θ(s_t) + 噪声（探索）
        2. 执行动作，获得奖励 r_t 和新状态 s_{t+1}
        3. 存储经验：(s_t, a_t, r_t, s_{t+1})
        4. 更新价值网络：使用 TD 误差（见上文"TD 学习"部分）更新 V_φ，使其更准确预测累积奖励
        5. 更新策略网络：使用优势函数 A = Q - V 更新 π_θ，增加好动作的概率
        6. s_t = s_{t+1}（进入下一状态）
```

其中：

- $\pi_\theta$ 表示参数为 $\theta$ 的策略网络
- $V_\phi$ 表示参数为 $\phi$ 的价值网络
- $a_t = \pi_\theta(s_t) + \text{噪声}$ 表示策略输出加上探索噪声
- $A = Q - V$ 表示优势函数

**关键理解**：

- **状态**提供了决策所需的所有信息（当前情况）
- **策略**根据状态直接输出动作（怎么做）
- **动作**改变环境，产生新状态（执行）
- **奖励**提供即时反馈（好不好）
- **价值函数**评估长期收益，指导策略改进（值不值得）

在这个例子中，如果价值函数 $V(s)$ 很准确，它知道"接近杯子但还没抓取"的状态价值是 50，"已经抓取杯子"的状态价值是 150，"到达目标位置"的状态价值是 200。但策略网络仍然必需，因为：

1. 动作空间是连续的（关节角度可以是任意值），无法枚举所有可能
2. 策略网络直接输出具体的动作值 $[\Delta\theta_1, \Delta\theta_2, ...]$
3. 价值函数只评估，不执行

这就是为什么 Actor-Critic 方法（如 DDPG、PPO）同时需要策略网络和价值网络：策略网络负责"执行"（输出动作），价值网络负责"评估"（指导改进）。

### On-Policy vs Off-Policy

这是强化学习中的两个重要概念，区别在于**用于学习的策略**和**用于收集数据的策略**是否相同：

#### On-Policy（同策略）

- **定义**：用于学习的策略（目标策略）和用于收集数据的策略（行为策略）是同一个策略
- **特点**：
  - 必须使用当前策略收集的数据进行学习
  - 学习过程中策略会不断更新，旧数据可能不再适用
  - 需要持续与环境交互获取新数据
- **典型算法**：PPO、A2C、TRPO

**实际例子 - PPO（Proximal Policy Optimization）**：PPO 是典型的 On-Policy 算法。在训练过程中，PPO 使用当前策略收集一批数据（如 1000 个时间步），然后用这些数据更新策略。更新后，旧数据就不再适用了（因为策略已经改变），必须丢弃旧数据，重新用新策略收集数据。这就像"边学边用"：学到的知识立即应用到实践中，但旧的经验不能重复使用。例如，在训练游戏 AI 时，PPO 每收集一批游戏数据就更新策略，然后必须重新玩游戏获取新数据，不能重复使用之前的游戏记录。

#### Off-Policy（异策略）

- **定义**：用于学习的策略和用于收集数据的策略可以不同
- **特点**：
  - 可以使用其他策略（如旧策略、随机策略）收集的数据进行学习
  - 可以重复利用历史数据，数据效率更高
  - 支持经验回放（Experience Replay）
- **典型算法**：DQN、DDPG、TD3、SAC

**实际例子 - DDPG（Deep Deterministic Policy Gradient）**：DDPG 是典型的 Off-Policy 算法。它使用经验回放缓冲区存储历史经验（可能来自不同时期的策略），然后随机采样这些历史数据进行学习。例如，在训练机器人控制任务时，DDPG 可以将过去 100 万步的经验都存储在缓冲区中，每次训练时随机采样 64 个样本，即使这些样本是 1 小时前用旧策略收集的，仍然可以用来训练当前策略。这就像"温故知新"：可以反复学习历史经验，数据利用率高，但需要处理新旧策略的分布差异。

**核心区别**：On-Policy 要求数据必须来自当前策略，而 Off-Policy 可以使用任何策略的数据，通过重要性采样等技术来修正分布差异。**PPO vs DDPG**：PPO 是 On-Policy 的代表，训练稳定但数据效率较低；DDPG 是 Off-Policy 的代表，数据效率高但需要仔细调参。两者在后续章节会详细介绍。

## 深度强化学习算法

### DDPG（Deep Deterministic Policy Gradient）

DDPG 是一种用于连续动作空间的 Off-Policy 算法，结合了 DQN 和策略梯度方法。

#### 核心思想

- **Actor-Critic 架构**：
  - **Actor**：确定性策略网络，直接输出动作值（而非动作概率分布）
  - **Critic**：Q 值网络，评估状态-动作对的价值

- **关键技术**：
  1. **经验回放（Experience Replay）**：存储历史经验，随机采样打破数据相关性
  2. **目标网络（Target Network）**：使用独立的目标网络计算 Q 值目标，提高训练稳定性
  3. **软更新（Soft Update）**：目标网络参数通过缓慢更新（如 θ_target = τ·θ + (1-τ)·θ_target）而非直接复制

#### 算法流程

```
1. 初始化 Actor 网络 μ(s|θ^μ) 和 Critic 网络 Q(s,a|θ^Q)
2. 初始化对应的目标网络，参数复制
3. 初始化经验回放缓冲区 R
4. For each episode:
   a. 初始化状态 s
   b. For each step:
      - Actor 选择动作 a = μ(s) + N（添加探索噪声）
      - 执行动作，获得奖励 r 和下一状态 s'
      - 存储 (s, a, r, s') 到 R
      - 从 R 中采样小批量样本
      - 更新 Critic：使用 TD 学习最小化 TD 误差（更新 Q 值网络）
      - 更新 Actor：最大化 Q 值（策略梯度）
      - 软更新目标网络
```

#### 适用场景

- 连续控制任务（如机器人控制、自动驾驶）
- 需要精确动作值的场景
- 可以离线收集数据的场景

### PPO（Proximal Policy Optimization）

PPO 是一种 On-Policy 算法，通过限制策略更新的幅度来保证训练稳定性。

#### 核心思想

- **问题**：策略梯度方法在更新时，如果步长过大，可能导致策略性能急剧下降
- **解决方案**：使用裁剪机制限制策略更新的幅度，确保新旧策略不会差异过大

#### 关键技术

1. **重要性采样比率**：

   ```
   r(θ) = π_θ(a|s) / π_θ_old(a|s)
   ```

   衡量新旧策略的差异

2. **裁剪目标函数**：

   ```
   L^CLIP(θ) = E[min(r(θ)·A, clip(r(θ), 1-ε, 1+ε)·A)]
   ```

   其中 A 是优势函数，ε 是裁剪参数（通常 0.1-0.3）

3. **优势函数估计**：通常使用 GAE（Generalized Advantage Estimation）来估计优势

#### 算法流程

```
1. 初始化策略网络 π_θ 和价值网络 V_φ
2. For each iteration:
   a. 使用当前策略收集一批轨迹数据
   b. 计算优势函数 A_t（使用 GAE）
   c. For K epochs:
      - 计算重要性采样比率 r(θ)
      - 更新策略：最大化裁剪目标函数
      - 更新价值函数：最小化价值函数误差
   d. 丢弃旧数据，重新收集
```

#### 优势

- **稳定性好**：裁剪机制防止策略更新过大
- **实现简单**：相比 TRPO 更容易实现和调参
- **样本效率**：可以多次使用同一批数据进行更新（K epochs）

#### 适用场景

- 离散和连续动作空间都适用
- 需要稳定训练的场景
- 可以持续与环境交互的场景

## 算法对比

| 特性       | DDPG                 | PPO                |
| ---------- | -------------------- | ------------------ |
| 策略类型   | Off-Policy           | On-Policy          |
| 动作空间   | 连续                 | 离散/连续          |
| 数据效率   | 高（可复用历史数据） | 中等（需新数据）   |
| 训练稳定性 | 中等（需调参）       | 高（裁剪机制）     |
| 实现复杂度 | 中等                 | 简单               |
| 典型应用   | 机器人控制、连续控制 | 游戏AI、通用RL任务 |

## 总结

- **On-Policy vs Off-Policy**：核心区别在于是否可以使用其他策略的数据进行学习
- **DDPG**：适合连续控制，数据效率高，但需要仔细调参
- **PPO**：通用性强，训练稳定，是目前最流行的 RL 算法之一

选择算法时，需要考虑任务特性（离散/连续动作）、数据获取方式（在线/离线）、以及训练稳定性要求。
